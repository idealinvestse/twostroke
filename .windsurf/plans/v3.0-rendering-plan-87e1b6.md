# Rendering Engine v3.0 Plan - Godot 3D

Migrate the 2-stroke engine simulation to Godot 4.x for professional 3D visualization with Vulkan rendering, real-time effects, and standalone Windows deployment.

## Executive Summary

**Platform:** Windows standalone executable  
**Quality Target:** Maximal - photorealistic 3D with PBR materials  
**Renderer:** Godot 4.x Forward+ (Vulkan) with 3D pipeline  
**Goal:** Full 3D engine visualization with animated components, dynamic lighting, particle systems, and professional dashboard UI

**User Requirements Confirmed:**
1. ✅ Windows platform
2. ✅ Maximal quality
3. ✅ 3D visualization
4. ✅ Single .exe export

**Current Pain Points in v2.1:**
- Bloom effect causes freezes (pixel-by-pixel CPU operations)
- No 3D capabilities
- Limited shader support
- Software rendering bottleneck

## Option A: Raylib Python (Alternative)

### Why Raylib?
- **Simple API:** Similar learning curve to PyGame
- **Hardware acceleration:** OpenGL backend with shaders
- **Python bindings:** `raylib-python-cffi` available on PyPI
- **Single-file dependency:** No massive engine overhead
- **Built-in effects:** Bloom, blur, post-processing shaders included

### Migration Strategy

#### Phase 1: Setup & Validation (1 week)
1. Install `raylib-python-cffi` and validate basic window creation
2. Port simple shape rendering (circles, rectangles, lines)
3. Benchmark: Ensure 60 FPS with 1000+ particles

#### Phase 2: Engine Rendering (2 weeks)
1. Port cylinder, piston, crankshaft geometry
2. Implement animated components with `DrawModel()` / `DrawMesh()`
3. Add real-time shadows and lighting
4. Port PV diagram rendering with `DrawLineStrip()`

#### Phase 3: Effects & Particles (1 week)
1. Enable hardware bloom (built into Raylib)
2. Port particle system to `DrawBillboard()` or `DrawMeshInstanced()`
3. Add combustion glow with point lights
4. Temperature-based particle shaders

#### Phase 4: UI & Gauges (1 week)
1. Port analog gauges using `DrawRectangleRounded()` + `DrawLineEx()`
2. Add real-time chart rendering with `rlImGui` or RayGUI
3. Implement quality presets using `SetShaderValue()`

#### Phase 5: Polish (1 week)
1. Optimize draw calls with `rlPushMatrix()`/`rlPopMatrix()`
2. Add save/load settings
3. Testing across Windows/Linux

### Technical Details

**Dependencies:**
```bash
pip install raylib-python-cffi
```

**Key API Changes:**
```python
# PyGame → Raylib
pygame.draw.circle() → DrawCircle()
pygame.draw.rect() → DrawRectangleRounded()
pygame.Surface → RenderTexture2D (for offscreen)
pygame.BLEND_RGBA_ADD → BlendMode blending with shaders
```

## Option B: Godot with Python (Alternative)

### Why Godot?
- **Full game engine:** Scene system, physics, animation
- **Python support:** Via PyGodot or GDScript (Python-like)
- **Node-based:** Easier to organize engine components
- **Built-in UI:** RichTextLabel, TextureProgress for gauges

### Migration Strategy

#### Phase 1: Godot Setup (3 days)
1. Download Godot 4.x engine
2. Create new project with 2D renderer
3. Set up GDExtension for Python or use GDScript

#### Phase 2: Scene Structure (1 week)
1. Create `Engine.tscn` with:
   - Cylinder (Polygon2D or Sprite2D)
   - Piston (RigidBody2D with custom draw)
   - Crankshaft (Node2D with rotation)
   - ParticleSystem (CPUParticles2D/GPUParticles2D)
2. Port physics calculations to `_physics_process()`

#### Phase 3: Rendering Pipeline (1 week)
1. Enable WorldEnvironment with glow/bloom
2. Add PointLight2D for combustion glow
3. Configure CanvasLayer for UI
4. Port PV diagrams to custom `Control` node

#### Phase 4: UI & Gauges (1 week)
1. Use Godot's built-in `TextureProgressBar` for RPM
2. Create custom gauge scene with `Polygon2D` needles
3. Real-time charts with `Line2D`
4. Settings panel with `OptionButton` for quality presets

#### Phase 5: Export (3 days)
1. Export Windows executable
2. Export HTML5 (WASM) for web demo
3. Performance profiling with Godot profiler
